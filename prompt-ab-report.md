## 5. [还没准备好] 拖延藏在准备背后

**用户北极星**：在 3 年内推出我自己的第一个 SaaS 产品

**用户输入**：我想做但感觉自己还没准备好，技能/资源/经验都还差点

### v1 输出

```
（先接住“还没准备好”这个感受）  
嗯，我听到你说“还没准备好”——这种感觉特别真实。很多人卡在这一步，总觉得要等一切就绪才能开始。  

（不直接给建议，而是用书里的视角轻轻点一下）  
Elon 在书里提到过一个类似的情境：他早期做 Zip2 的时候，其实也没有“准备好”。他当时连完整的编程经验都不够，但他做了一件事——**把问题拆解到能动手的最小单元**。比如，他先试着给本地商家做一个简单的在线展示页，而不是一上来就构建完整的 SaaS 平台。  

（把视角拉回用户身上，避免说教）  
你提到技能、资源、经验都差点……如果只选一个当下最能动手的“最小单元”，你觉得会是什么？
```

**v1 标签**：字数：295

### v2 输出

```
你不是没准备好，你是在等一个完美的开始。但完美开始不存在。埃隆做特斯拉时，电动车行业几乎为零。他缺的比你多。
```

**v2 标签**：字数：54
