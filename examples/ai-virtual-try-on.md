# AI 换装小程序

## 用户输入

```text
/product 我想做一个换装小程序，用户输入一张人物图和一张商品图，就可以换衣服。
```

## 普通 AI 容易回答

```text
可以先做 MVP，优化换装效果，找到目标用户，再做差异化。
```

## 一人公司.skill 商业判断卡

结论：不要先做泛 C 端娱乐换装，先做电商素材服务。

付费可能性：中。消费者可能会玩，但 API 成本很容易吃掉收入；商家为可发布素材付费的概率更高。

最可能付费的人：小红书服装商家、淘宝/抖音小店、私域卖货团队、电商代运营。

直接竞品：FASHN AI、VModel、Botika、VTry 这类虚拟试衣和 AI fashion model 产品。

替代方案：真人模特拍摄、P 图修图师、电商设计师、买家秀收集、通用 AI 生图工具。

可收费切口：商家给 5 张商品图，你交付 20 张商品上身图、详情页图或小红书种草图，定价 99/199/499 元。

第一单动作：今天找 10 个正在上新的服装商家，发 3 张样图，不讲技术，只问是否愿意为一组可发布素材付 99 元。

7 天验证：完成 3 个付费样单，记录单张图 API 成本、返工次数、商家是否发布。

停损线：10 个商家里没有 3 个愿意付 99 元，或每单返工超过 2 轮，就不要继续做小程序。

## 为什么更好

- 它没有把“换装效果”当成付费理由，而是把付费方从玩家转到商家。
- 它把 API 成本、返工、平台发布风险放进判断。
- 它先卖服务包验证支付，再决定是否产品化。
