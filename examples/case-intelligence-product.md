# 一人公司案例库

## 用户输入

```text
/pricing 我想做一个一人公司案例库。
```

## 普通 AI 容易回答

```text
可以做订阅制知识库，持续更新案例，吸引独立开发者付费。
```

## 一人公司.skill 商业判断卡

结论：案例本身不值钱，减少错误决策才值钱。

付费可能性：中。收藏案例是弱需求，做决策前的相似案例诊断更容易收费。

最可能付费的人：正在做产品选择、定价、获客或转型的一人公司创业者。

直接竞品：Trends.vc、Exploding Topics、FounderPal、商业案例库、付费情报报告。

替代方案：免费 GitHub awesome 列表、公众号文章、即刻帖子、播客、微信群讨论。

可收费切口：产品想法诊断报告、相似案例拆解、第一单路径设计、停损线设计。

第一单动作：不要先卖订阅。先卖 99/199 元单次产品判断报告，交付 3 个相似案例和 1 条验证路径。

7 天验证：找 20 个正在做产品的人，卖出 3 份诊断报告。

停损线：用户只收藏案例、不愿为具体判断付费，说明信息焦虑存在，但商业支付点不成立。

## 为什么更好

- 它把“案例数量”转成“减少误判”。
- 它避免一开始做重订阅产品。
- 它给出更容易成交的单次服务切口。
