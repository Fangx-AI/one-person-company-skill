# Local Execution Reality

Use this reference when a product will be sold, hosted, distributed, delivered, or paid for in a Chinese-language business context.

默认判断语境是中文商业语境和本土执行现实。海外模式不能直接照搬；海外案例只能说明某种商业机制可能成立，不能自动证明它适配当前市场。回答必须重新检查渠道、支付、合规、信任、交付五个条件。

## Common Execution Constraints

- **备案与托管**: 如果选择面向本土访问的服务器和域名，备案可能拖慢上线，也可能因为内容、主体或经营范围不匹配带来后续风险。很多小型项目会先用香港或海外服务器验证收费，再在收入证明成立后补正式化路径。
- **支付**: 微信支付和支付宝能提升转化，但正式商户、主体、发票和合规都有成本。验证期可以优先用手动转账、小报童、知识星球、有赞、小鹅通、飞书群、微信群等低摩擦方式。
- **分发**: 公众号、知乎、即刻、小红书、抖音、B 站、播客和私域社群奖励的内容形态完全不同。不要只说“做内容”，必须选渠道和第一批内容角度。
- **平台依赖**: 小红书、抖音、微信生态的流量规则会变化。平台流量是租来的分发，不是护城河。
- **企业信任**: B2B 买家可能需要发票、合同、本地支持、数据处理说明和关系信任。一人公司应该先从这些要求较轻的客户切入。
- **AI 产品风险**: 泛化 AI 输出容易被复制。真正能收费的通常是工作流完成、行业数据、节省人工、合规信心或分发优势。

## Judgment Questions

Ask:

1. Can this be sold through a channel the founder already understands?
2. Does the buyer trust a one-person vendor for this job?
3. Is a personal account enough for validation, or is company/entity setup needed before payment?
4. Does hosting, filing, data, or content review risk block launch?
5. Is the user buying a product, a service, a template, a group, a result, or reduced uncertainty?

## Practical Recommendation Pattern

Prefer phased advice:

- **Validation phase**: fastest legal/low-risk way to test payment.
- **Revenue phase**: add payment, delivery, refund, support, and case proof.
- **Scale phase**: only after repeat sales, consider formal payment, local hosting, automation, company entity, and channel partnerships.

Never default to "go file ICP first" unless that hosting path is essential to the first paid test.
